The Source

Canadian Internet Usage Survey from Statistics Canada


Target population

 * All persons 15 years of age and older living in the ten provinces of Canada.


Data Collection

 * Sample survey with a cross-sectional design
 * meaning multiple groups from different provinces, age groups, and household income quartiles were asked in similar proportions


Survey Design

 * Responding to this survey is voluntary.
 * Data are collected directly from survey respondents.
 * Data are collected through an electronic questionnaire or computer assisted telephone interviewing


The Limitations

Electronic Questionnaire to Evaluate Digital Access

 * Collecting data digitally could skew response volume towards digitally active users


Stratified Random Sampling

 * To address the possible skew, the data is collected from each group independently.
 * This keeps data volumes proportional by category and reduces variance

The Method: Multinomial Logistic Regression

How It Works

* Multionomial Logistic Regression is at heart a Classifer

* The model trains itself on existing data, in our case:
  * Internet Access
  * Online Shopping Metrics
  * Social Networking Usage

* From this data it identifies patterns in what the data LOOKS like when it comes from a certain category

* Once the model is trained, when new data is presented - in our case, the data would be the percentage of households in a geographical area that have internet access at home - the model can calculate a percent chance that that region is a low, second, third, or highest income quartile area

* It will then categorize that data - select the highest percentage category - and give that answer back to us

A Walkthrough

We will walk through the digital access model using Internet Access
The Goal

Use Multinomial logistic regression to create a model that takes the percentage of households in a geographical area that have internet access at home and calculates if that region is a low, second, third, or highest income quartile area

We use the Multinom function from the R package "nnet"

The multinom function trains the model based on the data provided in the scatter plot to the left

We know the model has trained itself adequately if it CONVERGES (meaning it has found a satisfying solution) within 100 iterations

R Code

We use this R code to generate a model, that says "predict household income quartile based on internet access data"

Weights

Using the summary function from R, we can see the weights the model has determined through training

The weights are a vital component to determining the percent chance a new data point fits in a certain category

Determining Accuracy: P-Values

P-Values are a metric used in statistic that tell the statistician:

"Is this model, given the data it was trained on, good enough to make valid predictions"
P-values are considered significant when they are below 0.05


This indicates that our model is accurate

Given these P-values we may say "There is a statistically significant correlation between digital access and income quartile"
